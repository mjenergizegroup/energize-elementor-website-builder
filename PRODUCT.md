# Product

## Register

product

## Users

The Energize Group web team, including technical and non-technical production
staff, uses this internal desktop tool to build and migrate dental practice
websites. Users work through repeatable production tasks and need to understand
the next action without interpreting WordPress, Elementor, or JSON internals.

## Product Purpose

Energize Website Builder turns reusable layouts, source-site content, brand
settings, and a WordPress destination into a clear website production workflow.
Success means a teammate can plan pages, reuse the right layout, import content,
review readiness, and create safe WordPress drafts with minimal manual handling
and no exposure to implementation details that do not affect their decision.

## Brand Personality

Precise, confident, practical. The product should feel like focused production
software used every day: fast to scan, direct in its language, and dependable
under repetitive workloads.

## Anti-references

Do not resemble a generic decorative SaaS dashboard, a developer console, or an
Elementor diagnostics screen. Avoid unexplained IDs, filenames, source-site
metadata, plugin ledgers, dependency terminology, ornamental motion, and visual
effects that compete with the task.

## Design Principles

1. Show decisions, hide implementation details.
2. Prefer one obvious next action over several technical choices.
3. Use plain language that a new teammate can understand without training.
4. Make reusable layouts visually recognizable before they are assigned.
5. Preserve safety and deterministic behavior behind a calm daily workflow.

## Accessibility & Inclusion

All daily workflows must support keyboard navigation, visible focus states,
screen-reader labels, and adequate contrast. State must never rely on color
alone. Motion should be brief, functional, and compatible with reduced-motion
preferences. Dense desktop layouts must retain readable labels and predictable
interaction order.
